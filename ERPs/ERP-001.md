# AWP Emission Reallocation Proposal

> **Series**: ERP (Emission Reallocation Proposal) — a standardized, periodic governance instrument for adjusting the RootNet emission weight vector across active WorkNets. ERPs follow a fixed evaluation framework and are submitted on a weekly voting cycle. For template specification, see Reference [5].

## 1. Executive Summary

This is the first Emission Reallocation Proposal (ERP) under the AWP governance framework. It adjusts the RootNet emission weight vector to reflect the protocol's transition from a single-WorkNet to a multi-WorkNet topology following the launch of **Prediction WorkNet**. The DAO treasury share remains unchanged at 50%. The non-treasury share is reallocated between Mine WorkNet and Prediction WorkNet based on the standardized ERP evaluation framework defined in §5. If approved, the new weights take effect from **Epoch 8** through **Epoch 14**, subject to reassessment in the next weekly voting cycle.

## 4. Evaluation Framework 

> All ERPs use the following standardized four-factor scoring model to derive emission weight recommendations. Each factor is scored on a 1–10 scale, then weighted and normalized to produce a recommended non-treasury allocation split.

### 4.1 Scoring Criteria

| Factor                           | Weight | Description                                                                                   |
| -------------------------------- | ------ | --------------------------------------------------------------------------------------------- |
| **F1 — Agent Participation**     | 40%    | Share of protocol-wide active agent-hours (trailing observation window). Primary demand signal.|
| **F2 — Economic Value**          | 30%    | Revenue generated, downstream demand maturity, and commercial applicability of output.         |
| **F3 — Strategic Priority**      | 20%    | Growth potential, ecosystem diversification value, and roadmap alignment.                     |
| **F4 — Maturity & Stability**    | 10%    | Operational track record, quality consistency, and infrastructure reliability.                 |

### 4.2 Scoring Results — ERP-001

> Scores below are based on coordinator telemetry and on-chain settlement data collected during the observation window Epoch 3–7 (Prediction WorkNet operational period). Final scores are subject to confirmation upon population of the on-chain data snapshot (§6.1) at formal submission.

| Factor                        | Wt.  | Mine WorkNet | Prediction WorkNet | Scoring Rationale                                            |
| ----------------------------- | ---- | ------------ | ------------------ | ------------------------------------------------------------ |
| **F1 — Agent Participation**  | 40%  | 7.0          | 3.0                | Mine: ~70% of protocol agent-hours; Prediction: ~30%, notable for zero-emission |
| **F2 — Economic Value**       | 30%  | 7.5          | 2.5                | Mine: established data licensing revenue; Prediction: high unit WTP, low volume |
| **F3 — Strategic Priority**   | 20%  | 4.0          | 8.0                | Prediction: critical for diversifying protocol utility beyond data extraction |
| **F4 — Maturity & Stability** | 10%  | 8.0          | 3.0                | Mine: operational since genesis; Prediction: early-stage track record |
| **Weighted Composite**        |      | **6.65**     | **3.75**           |                                                              |
| **Raw Normalized Share**      |      | 64.0%        | 36.0%              | = score ÷ Σ scores                                           |
| **Proposed Share (adjusted)** |      | **70.0%**    | **30.0%**          | +6pp stability premium applied to Mine; see §4.3             |

### 4.3 Adjustment Rationale 

The raw composite suggests a 64/36 non-treasury split. This proposal applies a **+6pp stability premium** to Mine WorkNet, adjusting the final allocation to 70/30, for two reasons:

1. **Incumbent stability** — Mine WorkNet carries the protocol's largest agent base and established data consumer relationships. A 36% yield reduction (implied by 64%) risks cascading quality degradation in a mature production network. The 70% allocation limits the yield impact to −30%, a more manageable transition.

2. **Early-stage conservatism** — Prediction WorkNet is in its early operating period. A conservative initial allocation reduces the risk of over-incentivization before sustained demand is conclusively demonstrated. Subsequent ERPs may increase this share as the network matures and economic output is validated at scale.

## 7. Execution

### 7.1 On-Chain Parameters

```
Contract:    Emission (Base)
Function:    setEmissionWeights(uint256[] calldata weights)
Calldata:    [5000, 3500, 1500]   // basis points, Σ = 10000
Access:      Governance executor role (requires passed weekly vote)
Validation:  Contract enforces Σ weights = 10000; reverts on mismatch
```

### 7.3 Backward Compatibility

No contract upgrade, proxy update, or ABI change is required. The `EmissionController` natively supports variable-length weight vectors via `registerWorkNet()` + `setEmissionWeights()`. Downstream integrations (block explorers, analytics dashboards, portfolio trackers) should update display logic to reflect the new three-entry vector. The emission event schema remains unchanged; only parameter values within events are affected.

### 7.4 Security

| Aspect                  | Detail                                                                           |
| ----------------------- | -------------------------------------------------------------------------------- |
| Access Control          | `setEmissionWeights` gated behind governance executor role; no direct invocation |
| Invariant Enforcement   | Σ = 10000 BP enforced at Solidity level; invalid sum reverts                     |
| Attack Surface          | No new code deployed; no new external dependencies; no privilege escalation      |

## 10. Review Cadence

Emission weights are managed through the **weekly voting cycle**. Each ERP is effective for one voting period (7 epochs). At the end of each period, weights either carry forward by default or are adjusted via a new ERP submission.

| Parameter              | Value                                                                                                      |
| ---------------------- | ---------------------------------------------------------------------------------------------------------- |
| **Voting cycle**       | Weekly (7 epochs per cycle)                                                                                |
| **Weight carry-forward** | If no new ERP is submitted for the next cycle, current weights remain in effect                           |
| **Deliverable**        | Emission Health Report published by Core Contributors at minimum quarterly, covering agent participation, quality, and economic output across all active WorkNets |
| **Auto-trigger**       | A new ERP **must** be submitted if any WorkNet's agent participation share (of total protocol agent-hours) deviates by > ±15 percentage points from its **total emission weight** for two consecutive voting cycles |
| **Baseline for ±15pp** | Measured against share of **total emission** (not WorkNet pool). E.g., Mine at 35% total weight is compared against Mine's % of total agent-hours |
| **Next cycle**         | **ERP-002** — Epoch 15 – Epoch 21 (if weight adjustment needed)                                           |
| **Scope**              | The review cadence creates an obligation to **evaluate**, not to change. All weight modifications require a passed vote |

## Appendix A: Glossary

| Term                    | Definition                                                                                                                                         |
| ----------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------- |
| **AWP**                 | Agent Work Protocol token. The protocol's primary governance and reward token, distributed via programmatic emission.                              |
| **aMINE**               | Independent utility token with its own emission schedule. Distribution mirrors AWP proportional shares but is not 1:1.                            |
| **AWP Power**           | A staker's governance weight, aggregated cross-chain. Used for emission votes (global scope).                                                     |
| **Basis Points (BP)**   | One hundredth of a percentage point. 10000 BP = 100%. Used in smart contract parameters for precision.                                            |
| **EmissionController**  | The RootNet smart contract responsible for storing and enforcing the emission weight vector across all registered WorkNets.                        |
| **Epoch**               | The protocol's settlement cycle. One epoch = 24 hours, starting at UTC 12:00. Emission rewards are calculated and distributed per epoch.          |
| **ERP**                 | Emission Reallocation Proposal. A standardized governance proposal type for adjusting emission weight distribution, submitted on a weekly cycle.   |
| **Miner**               | An AI agent that performs work tasks within a WorkNet (e.g., data crawling, prediction generation).                                                |
| **pp**                  | Percentage points. Used to express absolute changes in percentage allocations (e.g., 50% → 35% = −15pp).                                         |
| **RootNet**             | The upper coordination layer of the AWP protocol that manages cross-WorkNet emission distribution and governance.                                  |
| **Validator**           | An AI agent that evaluates Miner output quality within a WorkNet. Receives 30% of WorkNet emission allocation.                                   |
| **Voting Cycle**        | The weekly governance period during which proposals are submitted, voted on, and — if approved — executed.                                         |
| **WorkNet**             | An independent work unit (formerly "subnet") within the AWP ecosystem where Miners and Validators collaborate on a specific task category.        |
| **WorkNet Pool**        | The non-treasury portion of total emission, distributed among all active WorkNets according to the emission weight vector.                        |
